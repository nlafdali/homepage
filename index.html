Ablauf einer Bestellung innerhalb einer Microservices-Architektur

Das folgende Sequenzdiagramm zeigt, wie der Bestellservice mithilfe von Feign Clients mit dem Benutzer- und Produktservice kommuniziert. Der Bestellservice empfängt eine Anfrage vom Benutzer, ruft über Feign Clients die benötigten Daten aus dem Benutzerservice und dem Produktservice ab und speichert anschließend die Bestellung in seiner eigenen Datenbank.

  • Unabhängigkeit: Jeder Service verwaltet eigene Daten und Logik.
  • Kommunikation: Über HTTP/REST mittels OpenFeign.
  • Skalierbarkeit: Dienste können unabhängig entwickelt und bereitgestellt werden.

Formularsystem mit Spring Boot

Dieses Projekt zeigt ein Formularsystem, das mit Spring Boot und JPA entwickelt wurde. Benutzer können Formulare ausfüllen, speichern und abrufen. Die Daten werden über ein REST-API verwaltet und in einer Datenbank gespeichert.

  • Backend: Java Spring Boot mit JPA
  • Tests:
    • Integrationstests mit @SpringBootTest
    • Unit-Tests (Mockito)
  • Flyway: Flyway-Migrationen für DB-Versionierung
  • Ziel: Datenpersistenz, Validierung und REST-Kommunikation
